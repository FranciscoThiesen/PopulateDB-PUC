1 -
Relações são basicamente as tabelas do banco de dados, aonde temos um conjunto de 
entradas ( tuplas ) que consistem dos mesmos atributos.
Ex:
Carro

Atributos: São as "colunas" de cada tabela. Cada atributo admite um domínio de valores que tem que ser obedecido
por todas as tuplas.

Tuplas: São as entradas da tabela, ou em outras palavras um elemento do mundo real com valores válidos para 
os atributos da tabela

B - Esquema interno, externo e conceitual. Cada um significa algo diferente

C - Superchave é um conjunto de atributos que quando combinados permitem identificar cada tupla unicamente.
Chave candidata é uma superchave, que é mínima no sentido de que não podemos tirar nenhum dos atributos selecionados
de forma que os outros atributos remanescentes ainda formem uma superchave.
Chave primária é uma chave a ser escolhida dentre as chaves candidatas. Ela será usada para manter a integridade
referencial

D - Integridade referencial -> Integridade garantida no sentido de não possuir chaves estrangeiras que não existam
em outra tabela como chave primária. Integridade semântica -> Todos os valores das tuplas tem que estar no domínio
de seus respectivos atributos, (uma garantia de certa forma que estamos guardando dados com uma semântica coerente)
